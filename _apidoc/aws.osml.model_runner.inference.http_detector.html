class aws.osml.model_runner.inference.http_detector.HTTPDetector(endpoint: str, name: str | None = None, retry: Retry | None = None)[source]

Bases: Detector

HTTPDetector is responsible for invoking HTTP-based model endpoints to run model inference.

This class interacts with a model endpoint over HTTP to send a payload for feature detection and retrieves geojson-formatted feature detection results. It supports retry logic with exponential backoff for network-related issues.

property mode: ModelInvokeMode

Defines the invocation mode for the detector as HTTP endpoint.

Returns:

ModelInvokeMode.HTTP_ENDPOINT

find_features(payload: BufferedReader, metrics: MetricsLogger) → FeatureCollection[source]

Invokes the HTTP model endpoint to detect features from the given payload.

This method sends a payload to the HTTP model endpoint and retrieves feature detection results in the form of a geojson FeatureCollection. If configured, it logs metrics about the invocation process.

Parameters:

  • payload – BufferedReader = The data to be sent to the HTTP model for feature detection.

  • metrics – MetricsLogger = The metrics logger to capture system performance and log metrics.

Returns:

FeatureCollection = A geojson FeatureCollection containing the detected features.

Raises:

  • RetryError – Raised if the request fails after retries.

  • MaxRetryError – Raised if the maximum retry attempts are reached.

  • JSONDecodeError – Raised if there is an error decoding the model’s response.
