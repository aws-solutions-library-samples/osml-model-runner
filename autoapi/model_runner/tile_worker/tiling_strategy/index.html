TilingStrategy defines an abstract interface for the code that determines how a large image is broken down into regions and tiles.

Identify the regions that should be created from this image.

Parameters:

  • processing_bounds – the bounds of the full image or area of interest in pixels ((r, c), (w, h))

  • region_size – the size of the regions in pixels (w, h)

  • tile_size – the size of the tiles in pixels (w, y)

  • overlap – the amount of overlap (w, h)

abstract cleanup_duplicate_features(processing_bounds: model_runner.common.ImageRegion, region_size: model_runner.common.ImageDimensions, tile_size: model_runner.common.ImageDimensions, overlap: model_runner.common.ImageDimensions, features: List[geojson.Feature], feature_selector: model_runner.inference.FeatureSelector) → List[geojson.Feature]

This method handles cleaning up duplicates caused by tiling by applying the feature selector to any features that come from overlap regions.

Parameters:

  • processing_bounds – the bounds of the full image or area of interest in pixels ((r, c), (w, h))

  • region_size – the size of the regions in pixels (w, h)

  • tile_size – the size of the tiles in pixels (w, y)

  • overlap – the amount of overlap (w, h)

  • features – the collection of features to deduplicate

  • feature_selector – the algorithm that will be used to resolve duplicates

Returns:

the collection of features with duplicates removed

model_runner.tile_worker.tiling_strategy.generate_crops(region: model_runner.common.ImageRegion, chip_size: model_runner.common.ImageDimensions, overlap: model_runner.common.ImageDimensions, only_full_tiles: bool = False) → List[model_runner.common.ImageRegion]

Yields a list of overlapping chip bounding boxes for the given region. Chips will start in the upper left corner of the region (i.e. region[0][0], region[0][1]) and will be spaced such that they have the specified horizontal and vertical overlap.

Parameters:

  • region – a tuple for the bounding box of the region ((ul_r, ul_c), (width, height))

  • chip_size – a tuple for the chip dimensions (width, height)

  • overlap – a tuple for the overlap (width, height)

  • only_full_tiles – true if we only want to generate tiles that match the chip_size

Returns:

an iterable list of tuples for the chip bounding boxes [((ul_r, ul_c), (w, h)), …]
